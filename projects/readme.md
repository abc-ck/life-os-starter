# projects

**Bottom line:** Time-bound personal outcomes. The starter contains no assumed projects; onboarding lists only real active work and parks everything else safely.

**When to read this:** Start here for non-business project status or when deciding whether an idea deserves an active project.

Each active project should have one clear status/readme file. Completed or parked projects move to a dated archive folder.
