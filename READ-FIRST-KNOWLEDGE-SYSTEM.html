A better memory system for AI

Why a few maps, summaries, and honest labels help an AI find the right knowledge more reliably than a giant web of backlinks.

The simple version: backlinks tell you that two notes are connected. This system also tells the AI where to begin, which file is current, which file is authoritative, and which old file should be ignored.

Backlinks are useful, but they do a different job

Obsidian backlinks and the graph are excellent for exploring ideas. They help a person wander from one related note to another. But an AI usually begins with filenames, folder maps, and text search. A dense graph does not automatically tell it which note is correct.

What we added

We did not create a complicated database. We made ordinary Markdown easier for agents to understand.

1

Task maps

Short directions such as: for this task, read these two files in this order.

2

Plain-English summaries

Each important note says its bottom line and when an agent should read it.

3

Small, controlled labels

Status, authority, source, updated, and reviewed answer the questions that matter.

4

Currency checks

Old notes are marked superseded or archived instead of quietly pretending to be current.

5

Folder indexes

Large folders get a one-line directory so an agent can choose before opening everything.

6

Automated validation

A checker catches broken links, missing labels, unindexed files, and misleading dates.

Where the improvement came from

  • Old plans stopped masquerading as current plans. Replaced files pointed directly to the living source.
  • Draft research stopped outranking approved decisions. Authority labels made the order clear.
  • Agents opened fewer wrong files first. Maps routed them to a small starting set.
  • Agents could reject a file from its first few lines. They no longer had to skim an entire note to discover it was irrelevant.
  • Answers showed where they came from. Provenance became part of retrieval instead of an afterthought.

Was it faster?

It required less retrieval work. The new routes usually named one to three target files, and the full tests averaged 5.7 files opened per task after including startup and supporting context.

We did not record wall-clock seconds for the old and new systems, so it would be dishonest to claim something like "twice as fast." The measured gain is fewer wrong turns, better source selection, and far less correction after an agent uses stale information.

Testing note: The original baseline was reconstructed from the actual pre-system Git version rather than run live. Each task had one trial, so this is a practical acceptance test, not a laboratory study. A second post-fix run scored 97%, with zero correctness failures again, suggesting the result was stable rather than a one-off.

Why we did not add hundreds of tags

More labels are useful only when they answer a repeated question. Decorative tags create maintenance work and can make retrieval less trustworthy when people stop updating them.

The chosen labels stay small because they answer the five questions an agent actually needs:

  • What kind of note is this?
  • Is it current?
  • How much authority does it have?
  • Did the owner approve it or did an AI draft it?
  • When did the information change, and when was it last checked?

Backlinks still belong where a human would genuinely want to jump between ideas. They support the system; they do not replace its index.

The result

This is not a bigger knowledge base. It is a knowledge base with clearer road signs. An AI can enter with less context, find the right shelf, recognize the current answer, and leave a trail another agent can follow.

The lasting value: the memory lives in ordinary files, so Claude, Codex, or a future agent can use the same system without rebuilding your history inside every new chat.
