# agent startup

## purpose

Orient an agent without reading the whole vault or restarting established work.

## trigger

Start of substantial work, a new thread, context loss, or a handoff.

## steps

1. Read `os/me.md`, `os/agent-rules.md`, `os/vault-map.md`, and `knowledge-map.md`.
2. Identify the relevant project/area and local instructions.
3. Load only the current status, spec, or handoff needed.
4. State working context in three bullets or fewer.
5. Name the next action before substantial work.

## boundaries

- Do not load deeper personal context unless materially relevant.
- Do not rewrite startup files without owner permission.
- Prefer continuity over re-architecture.
