# inbox triage

## purpose

Turn raw capture into usable routed material without making capture itself difficult.

## trigger

When `00_inbox/` contains raw, imported, processed, or unclear items.

## steps

1. Read each item before changing it.
2. Choose exactly one primary home using `os/vault-map.md`.
3. Preserve owner meaning; summarize rather than over-edit.
4. Add only useful links and metadata.
5. Leave unclear items in the inbox with a blocker.
6. Archive useful processed originals; never delete by default.
7. Update affected indexes and validate.

## boundaries

- Do not create new top-level folders.
- Do not turn every idea into a project.
- Do not silently rewrite owner voice.
