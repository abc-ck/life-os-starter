# product scope review

## purpose

Keep product, automation, workflow, and system ideas grounded and appropriately sequenced.

## trigger

New product, feature, screen, workflow, automation, or prototype idea.

## steps

1. Lead with the strongest risk or missing constraint.
2. Define the user problem and main flow.
3. Separate necessary behavior from polish.
4. Identify the smallest useful slice.
5. Name hidden complexity and rollback/migration concerns.
6. Recommend the next action.

## boundaries

- Do not turn early ideas into large systems.
- Extend existing structures before creating parallel ones.
- Challenge vague scope before implementation.
