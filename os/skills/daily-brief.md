# daily brief

## purpose

Create a small owner-fit start-of-day/work-block plan that makes the next action obvious.

## trigger

Morning, start of work, or when the owner asks what to focus on.

## steps

1. Check prior carry-forward, current status, deadlines, and constraints.
2. Use the owner's energy/attention preferences from `os/me.md`.
3. Select no more priorities than the owner approved.
4. Give each priority one concrete next action.
5. Keep optional ideas outside the main plan.

## outputs

- brief chat plan
- optional `journal/daily/YYYY-MM-DD.md`

## boundaries

- Do not write a motivational essay.
- Do not overload low-capacity days.
- Do not fabricate calendar or task information.
