# skill map

**Bottom line:** Registry of repeatable agent routines. Setup and metadata audit ship as core methods; every other routine remains a draft until the owner approves it through real use.

**When to read this:** Read when a task may match a repeatable workflow, then open only the relevant skill.

| skill | trigger | output |
|---|---|---|
| [[system-setup]] | first setup or major owner-approved redesign | approved personalized vault |
| [[metadata-audit]] | import, restructure, or retrieval drift | trustworthy metadata and maps |
| [[agent-startup]] | substantial new session or context loss | compact context and next action |
| [[inbox-triage]] | raw or unclear capture exists | routed material and blockers |
| [[daily-brief]] | start of day/work block | small owner-fit plan |
| [[eod-wrap]] | end of work block/day | record, repository checkpoints, next seed |
| [[weekly-review]] | weekly reset | progress, drift, and next priorities |
| [[research-synthesis]] | source-heavy or current question | decision-ready synthesis |
| [[product-scope-review]] | product/workflow/system idea | scope critique and next slice |
| [[read-first-code-review]] | code review or audit | evidence-backed findings |
| [[project-handoff]] | pausing meaningful work | concise continuity record |

Approve, revise, or remove draft skills during onboarding. Do not keep routines merely because the starter includes them.
