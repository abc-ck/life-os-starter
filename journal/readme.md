# journal

**Bottom line:** Chronological layer for daily records, weekly/monthly reviews, and durable decisions. It records what happened rather than acting as the main task manager.

**When to read this:** Read when reconstructing events, reviewing a period, or checking why a durable choice was made.

- `daily/` — `YYYY-MM-DD.md`
- `weekly/` — `YYYY-Www.md`
- `monthly/` — `YYYY-MM.md`
- `decisions.md` — newest-first durable decision log
